# AGENTS.md

## Read When

- 阅读、理解或分析这个仓库时
- 回答与仓库实现、文档、命令行为相关的问题时

## Project Rule

- 在阅读和理解项目时，如果发现文档、注释、命令说明或项目描述与当前实现不一致，必须主动向用户明确说明这个错误或过时之处。
- 说明时优先给出：
  - 错误说法是什么
  - 当前更准确的说法是什么
  - 结论依据来自哪些本地代码、文档或可验证来源
- 如果该错误会影响用户当前决策、操作步骤或理解路径，应优先指出，不要等用户追问。

## Example

- 如果总 CLI 文档仍写 `channels login` 是 “interactive channel login (WhatsApp Web only)”，但插件文档和实现已经表明其他支持登录流程的插件渠道也会复用这条命令，应主动指出该描述已不完整，并解释当前实际行为。
